**4. KPPV en rgression**
4.1 Rgression avec k fixe

  • Estimation des valeurs
    Dans le cadre de la rgression, l'objectif est de trouver le nombre de plus proche voisins (k) qui permet de "s'approcher" au mieux des valeurs observes dela variable chauff.
    Comme nous l'avons vu dans le cours, Deux tapes sont ncessaires :
    • La phase d'entrainement ou l'on effectue le paramtrage (on fixe le k) sur un chantillon du jeu de donnes (70%)
    • Puis la phase de validation (30%) o l'on value si le k fix en entrainement estime correctement les valeurs observes (annotes) sur un autre chantillon du jeux de donnes (cf. shma 1)
    • par dfaut, nous fixons le nombre de k plus proches voisins = 20

4.2 Rgression avec diffrentes valeurs de k

  • Jusqu' prsent, nous avons fix le nombre de plus proches voisins (=20) et nous obtenons une valeur de MSE qui reflte une certaine qualit de l'ajustement. La question que l'on peut se poser est alors la suivante : Peut on amliorer la qualit de l'ajustement en modifiant le nombre de k ?

  • Pour y rpondre, nous allons incrmenter le nombre de k et calculer, pour chaque incrment de k, l'erreur Quadratique moyenne.

In [39]:
kmax = 80
#---> On stocke dans un array les erreurs quadratiques moyennes
error_Reg =  np.zeros(kmax)
#--> On fait varier les valeurs de k = 1  kmax par pas de 1
kppv = np.arange(1,kmax+1,1)
#--> boucle 
for k in kppv:
    KNN_Reg = KNeighborsRegressor(n_neighbors = k)
    KNN_Reg.fit(X_train, Y_train)
    Y_pred = KNN_Reg.predict(X_test)
    #--> calcul MSE
    MSE = np.mean((Y_test - Y_pred)**2)
    #--> et stockage dans la liste
    error_Reg[k-1] = MSE
  • Nous construisons le graphique MSE = f(k) et nous calculons le k optimal

df_result_Reg = pd.DataFrame({
